4. API reference

class pysdg.synth.generate.Generator(gen_name: str, num_cores: int | None = None, work_dir: str | None = None, log_filename: str | None = None)

A class for synthetic data generation.

Parameters:

  • gen_name – The name of the generator.

  • num_cores – The maximum number of cores. If None, all available cores will be used.

  • work_dir – User-defined work directory to which some processes may save temprary files and directories. The default is to use current working directory.

  • log_filename – A name for the log file that to be created in work_dir. If None, logging to file will be disabled.

load(raw_data: str | DataFrame, raw_info: str | dict) → DataFrame

Safely loads the input dataframe and prepares it for training the generative model.

Parameters:

  • raw_data (str or pd.DataFrame) –

    • If a string, it should the full path to the input real dataset csv file including extension.

    • If a pandas DataFrame, it should be the Pandas dataframe of thre real data.

  • raw_info (str or dict) –

    • If a string, it should be the full path to the json file describing the real data including its extension.

    • If a dictionary, it should be the dictionary describing the real data.

Returns:

The loaded real data with all missing values properly processed.

unload() → list

Safely unloads the generated encoded synthetic dataset versions (synths).

Returns:

List of ‘synths’. All ‘synths’ have the same number of records and matching variable types od ‘real’.

train(input_data: DataFrame | None = None)

Trains the generator using the encoded real or the input dataset. To avoid training errors, make sure to use the load method first.

Parameters:

input_data – A pandas dataframe that is used to train the model. If passed, it should be a subset of encoded real dataset.

gen(num_rows: int | None = None, num_synths: int | None = None, no_obsvs: int | None = None, no_synths: int | None = None) → list

Generates multiple synthetic datasets (synths) from the trained generative model.

Parameters:

  • num_rows – The target number of required records (observations) in the output synthetic data.

  • num_synths – The target number of required synthetic versions (synths) where each ‘synth’ has the same number of the target no_obsv.

  • no_obsvs – Deprecated. Use num_rows instead.

  • no_synths – Deprecated. Use num_synths instead.

class pysdg.synth.optimize.BayesianOptimizationRoutine(gen: Generator, eval_function: callable, objective: str = 'maximize', n_trials: int = 10, study_name: str = 'my_study', dump_sqlite: bool = False, dump_csv: bool = False)

A class for Bayesian optimization of synthetic data generation.

Parameters:

  • gen – The pysdg generator object created for “synthcity_ctgan”.

  • eval_function – A callable function to evaluate the generator.

  • objective – The optimization objective, either “maximize” or “minimize”. Default is “maximize”.

  • n_trials – The number of optimization trials. Default is 10.

  • study_name – The name of the study. Default is “my_study”.

  • dump_sqlite – Whether to dump the study results to an SQLite database. Default is False. If True, the SQLite database will be saved as “study_name.db”.

  • dump_csv – Whether to dump the study results to a CSV file. Default is False. Enter the file path to dump the results.

pysdg.synth.metrics.combine_estimates(ests: List[Tuple], r=1) → Tuple

Calculates the combined estimate for a group of synthetic datasets (i.e. synths) of a same size that were generated by the same synthetic model.

Parameters:

  • ests – A list of the tuples of estimates and the corresponding standard errors for the synthetic synths.

  • r – The ratio k/n, where k is the sample size of the synthetic data (synth) and n is the size of the original data (real).

Returns:

A tuple of combined estimate and combined standard error.

pysdg.synth.metrics.compare_estimates(gt_vec: Tuple, syn_vec: Tuple, h0_value=0, alpha=0.05) → ndarray

Compares the decisions based on real and synthetic data and calculates their confidence interval overlap.

Parameters:

  • gt_vec – A tuple of ground truth estimate and its standard error.

  • syn_vec – A tuple of synthetic estimate and it standard error.

  • h0_value – The specific value that is being tested against. This is typically equal to zero (no effect) when testing regression coefficients.

  • alpha – Significance level.

Return: An array with follwing values:

Estimate agreement: A binary indicator for whether the estimate produced by the real data is within the CI produced by the synthetic data Decision agreement: A binary indicating whether the same conclusion is drawn in the real and synthetic estimates. So the estimates agree if the CI is insignificant in the real data and synthetic or if there is a positive significant effect in both or a negative significant effect in both. CI overlap: A float indicating what proportion of the real and synthetic CIs overlap. Standardized difference indicator: A binary indicator of whether the difference in the parameter estimate is consistent with the null hypothesis of no difference. A value of 1 indicates that there is no significant difference. Z value: The Z value for the input vectors. The value is used to compute the standardized difference indicator.

pysdg.privacy.mmbrshp.calc_mmbrshp_risk(synth: DataFrame, real_train: DataFrame, real_holdout: DataFrame, population_size: int, m: int | None = None, quasiID: list | None = None, h: int = 5, no_bins: int = 20) → Tuple[float, float]

Calculates the membership disclosure risk.

Parameters:

  • synth – The synthetic dataset generated by a generative model.

  • real_train – The real dataset that was sampled form the population and used in training a generative model.

  • real_holdout – The holdout dataset that was sampled form the population but not used in training a generative model.

  • population_size – The population size.

  • quasiID – The quasi variables of interest. All variables are used by default.

  • h – An integer representing the hamming distance threshold used in calculation.

  • no_bins – Number of bins used to discretize continuous and high cardinality discrete variables.

Returns:

Relative F1 score for membership disclosure risk. Naive F1 score for membership disclosure risk.

class pysdg.privacy.mmbrshp.MmbrshpRsk(real_data: DataFrame, population_size: int, h=10, seed=13, max_no_cores=None)

Calculates the membership disclosure risk associated with synthetic data.

Parameters:

  • real_data (dataframe) – Real Dataset as Pandas dataframe.

  • population_size (int) – The population size of from which the real dataset was sampled.

  • h (int) – An integer representing the hamming distance threshold to be used when calculating the membership disclosure risk.

  • seed (int) – An integer for fixing the seed to ensure reproducibility. Set to None if randomness is desired.

  • max_no_cores (str) – Specifies the number of cores as either ‘hi’ for maximum number of cores, ‘lo’ for 5 cores or None for no multiprocessing.
